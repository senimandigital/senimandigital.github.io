magic.js

Kita sering bingung untuk memilih menggunakan salah satu dari Frontend Framework yang sedang populer, jika kita melihat lebih dalam manapun Framework yang kita pilih, Semua-nya beresiko usang. Kenapa Framework Bisa Usang adalah karena feature yang disediakan oleh framework secara perlahan tapi pasti akan di akusisi oleh Bahasa Pemrograman.

Apabila kita biasa menggunakan bahasa pemrograman PHP, maka kita dapat hanya mempelajari penggunaan salah satu Framework karena PHP sudah sampai pada satu titik dimana kita bisa membuat component yang bekerja secara lintas Framework.

Kondisi luarbiasa lain-nya adalah, semua Framework dari berbagai bahasa pemrograman Backend bergerak ke arah yang sama, ini artinya aplikasi yang kita tulis menggunakan framework PHP dapat kita konversi dengan mudah secara agar dapat bekerja pada bahasa pemrograman lain dan proses konversi tersebut dapat kita lakukan manual menggunakan tool Bulk Find dan Replace.

Kenversi ini menjadi dimungkinkan dapat dilakukan dengan mudah, tentu tidak terlepas dari fakta bahwa para penggiat Opensource membuat Framework alias dari Framework PHP pada bahasa pemrograman lain.

Selain konstribusi yang diberikan penggiat Opensource dalam membuat Framework alias pada bahasa pemrograman yang berbeda, faktor penunjang penting lain-nya adalah, ada banyak juga konstributor yang membuat Library Subset dari berbagai Framework Populer, dengan banyaknya Library Subset yang tersedia, orang awam sekalipun dapat menyusun library untuk membentuk Framework yang baru dengan mengabungkan berbagai keunggulan sfesifik dari masing-masing Framework.
